App Name: TimeLord

Purpose:

Help people remember things that they need to accomplish and help them with time management skills

Need:

Naturally humans are not retentive creatures and therefore have trouble remembering everyday tasks and keeping on track with their schedule

Solution:

Create a mobile itinerary for users to write down what things they need to complete. Track these tasks and fire events when tasks is scheduled. Keep track if user checks in or not and as actually committing to task to provide accurate stats on how they are doing. Build auto-time allocation feature that corrected schedule to make sure user does not fall behind on their goal.

Obstacles Encountered:

Creating a mobile itinerary was the easiest implementation to achieve. Everything else relied heavily on a lot of logic and phone notification events which I was not ready to handle. For example, I wanted the itinerary to be sorted based on the time and dates scheduled, but I for the life of me could not come up with an effective compare function in order to sort the list of to-dos. In addition, having no knowledge on how to implement phone notification events my solutions such as the CheckIn, Event statistics, and Auto-Time allocation features would be difficult because I wouldn't know how to interact and manipulate the data for each task. Not only this, but even the simplest task was a bit challenging because even though I could display all the data that users were entering, all of it would disappear as soon as the closed.

Solutions for Obstacles:

I honestly tried to research and find methods on how to implement all these extra features for time management, but I felt it was best to go simple and at least achieve a mobile itinerary that would save all the content you entered. So in order, to do so, I created a function using HTML5 localStorage in order to dynamically create an array in JavaScript and push whatever values that were inputted to that array. Then I simply created a show function to display it all in a list. Each task that was added to the list had the date it was scheduled for and Remove button incase the user had finished.

Conclusions:

Overall, I really liked this assignment and even though I wasn't able to accomplish everything I stated in my prototype I still had fun and came close making what I originally designed. This is definitely not the final version of this app; and I do intend to get better so I can implement these additional features.
